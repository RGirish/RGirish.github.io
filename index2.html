An Education Evaluation System for Android

eyeObserve


September - November 2015


This project was developed as part of my part-time job at ASU's Mary Lou Fulton Teachers College. It is basically an education evaluation system that allows various levels of users in an educational system like schools or universities to interact with each other, evaluate each other's performance and such. The application connects with a no-SQL database at Parse. The application also allows capturing and uploading videos to an Ubuntu server at Digital Ocean.

My Contribution

I was the sole developer in this project. I coded all the client and server side modules. I designed the backend data structure at Parse and designed the control and logic flow in the system's architecture. I setup and maintained a Windows 2012 server for a brief period of time and then migrated it to an Ubuntu server later. I coded the PHP scripts needed to upload videos to the server too. I also developed a library in parallel, that allows any user to create forms dynamically, much like Google Forms. At present, I'm working on making this library accessible to the public.

Client-side: XML(UI Development), Java (Logic)
Server-side: Parse (no-SQL Database), Windows Server 2012, Digital Ocean's Linux Server - Ubuntu v12.x, PHP.

Secure Banking System

Sun Devil Bank


August - November 2015


Sun Devil Bank, a Secure Banking System was developed as part of the course project for CSE 545 - Software Security. The Secure Banking System is a system developed primarily to facilitate such secure banking transactions and user account management through the Internet. A banking organization often needs to track various operations performed by both the internal and external users using the organization’s banking infrastructure. The course being Software Security, the ultimate goal of this project was to develop a system that is secure in all ways, more than anything else. A greater importance had to be given to the security aspects and requirements of the system than the functional aspects and requirements.

My Contribution

I have served as the Group Leader for the course of this project. I was in-charge of all coordination among my team of 9 members. It was my responsibility to keep track of all the deadlines issued by the course instructor regarding the progress of the project. As a leader, I created and maintained a schedule of all activities to be done and a record of all activities that have been done by each of the group members. I divided the tasks to be done and allocated each member of the team, task to do.

As for the software system development, I first contributed to the construction of the SRS document. I developed the backend database structure and schema. I set up the database online, created and maintained it through the course of the project. In the next phase of the project, I took up the OTP (One Time Password Module and the Government User Module) and developed these alone. In the last phases of the the project, I helped the other members in testing the system for bugs and vulnearbilities and in the final deployment of the system to the server .

Languages/Frameworks used: Java, JSP, Spring Framework, Spring Security, Apache tomcat, MySQL, HTML, JSTL, CSS, JavaScript.

Computer Systems Security Projects


August - November 2015


This was a group of 4 small projects that I did as part of the course CSE598 - Computer Systems Security.

The first project was about experimenting with Buffer Overflows in C. I learnt about coding vulnerabilities and how an attacker can make use of them to intrude into the software system and gain unauthorized access to resources and memory.

The second project was a demo/learning experience on rainbow tables. I learnt to use tools like ophcrack to crack passwords for user accounts in a Windows XP PC.

The third project was about understanding Data Formats and memory structures. I learnt about the various data formats and how they're stored as 1s and 0s in the memory and what happens when we read one format as another and stuff like that. I understood the importance and nuances of memory and data formats and how an attacker can play with them.

The fourth project was about Sidejacking - stealing session cookies. There were 2 parts to this project. In the first part, I learnt to use Cookie Managers in a browser to copy the contents of cookies and steal a person's session. In the second part, I learnt to use network sniffers in capturing packets (cookies) and stealing the session over a wireless network.

Languages used: Mainly C.

Virtual Reality System for Architectural and Real Estate Projects

Project Unreal Estate


December 2014 - June 2015


I developed Unreal Estate as part of my internship at Samskrut Technologies. This is a Virtual Reality system for experiencing virtual yours of architectural and real estate projects. This projects consists of 2 applications - one with the VR experience, one without. The former works with VR headsets like the Google Cardboard. The app, along with the VR headset, when wrapped around your eyes, you can virtually view buildings and constructions. The latter is a simpler version of the same, which doesn't require a VR headset - instead, you can browse through the architectural projects either by swiping 360 degree panoramic images or by moving your phone in space.


My Contribution



I developed the front-end with XML and Java, with all the views on screen, compatible with the VR headset.

I designed the data structure at the back-end, and developed the back-end database in Parse - a non-SQL Database [www.parse.com].

I developed the functional architecture of the system, along with the base logic behind the system's working.

I developed and managed the web services required to connect the system with the server.

I designed the system to work offline too, by designing and creating a local database system within the Android device using SQLite.

I developed the modules that render Photosphere images and 360-videos in a split side-by-side view for the VR headset, using Panframe and Rajawali Libraries.

I partly worked on a module that uses OpenGL to modify and insert Hotspots inside the 360 view of images and videos.

Secure Banking System (CSE545)
The Secure Banking System (SBS) was the course project for the course Software Security. It is a system developed with Security as the most important requirement. It was developed in Java with the Spring and Spring Security Frameworks, using Apache Tomcat for the server. The goal of the project was to implement a system that could be accessed by any user of a bank with a browser and an internet connection.
