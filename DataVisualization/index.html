An Introduction

Our project is a visual analytics on the Stackoverflow Java dataset. Basically, what we have build is a visual comparison tool that can be used to compare 2 or more different correlation algorithms. In the context of our dataset, what correlation means is how closely or loosely 2 different tags are connected. For instance, Android and XML are 2 tags that are very closely related, but Android and Spring might not be as related as Android and XML.

Our tool places visualizations like Chord diagrams and Network visualizations side by side to compare the difference in the algorithms employed. The first algorithm that we have used is basic Text Mining (done in R). The second algorithm is something that we came up with ourselves. It basically goes like this: ‘If a user has contributed a lot to 2 different tags, then they’re probably very related to each other’. So, according to this algorithm, if 2 tags are shown as highly related, then they have a lot of common users who have contributed to these 2 tags.

Generally, we can replace these 2 algorithms with anything that can produce a correlation matrix between tags. This makes the tool customizable and dynamic.

Research Questions we have tried to answer:

  • Can the correlation between tags be analyzed from novel perspective apart from the traditional text mining algorithm?
  • Can we successfully model the correlation between tags using a new promising approach?
  • How tags are analyzed across different correlation algorithms ? How close are the tags interrelated with each other and how are they clustered differently?
  • Can we model a correlation between popular tags based on users’ contribution activities in Stack Overflow?

Methodology:

User Correlation Algorithm - This algorithm essentially frames the correlation matrix by analyzing the user interaction on various tags. The reason for choosing this algorithm is because a user always plays a vital role in any question and answer website. As per this algorithm, two tags A and B are said to be highly correlated if the two tags have a number of users in common. This commonality is obtained by running queries across various tags in the MySQL database.

R Text Mining (TM) Algorithm - This was the second algorithm used to find highly correlated tags and their relationship with other tags. A correlation matrix was constructed between the tags to find the highly correlated tag K-means clustering was used to cluster tags based on the value of the correlation coefficient and the correlation was represented as edge between two tags using the Euclidean distance value.

Data Collection:

  • Stack Overflow
  • Sampled year 2014 of forum posts in topic Java from StackOverflow.com {195370 questions, 167649 answers, 86672 accepted answers}

Findings:

  • A highly positive correlation was recorded between the eclipse and android.
  • Surprisingly android, arrays and regex were the most popular java tags.
  • Network visualization was effective in finding the closely related tags through clustering while chord diagram exhibited dependency among tags.
  • This visual analytics can be used for comparing any two different algorithms in the future.
